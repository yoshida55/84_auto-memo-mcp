# 📋 auto-memo補助MCPツール 仕様書

---

## 🎯 目的

Claude Code のスキル（`/memo`, `/memo-all`, `/memo-format` など）からメモを書くとき、  
現状は Claude が1ステップずつファイル操作しているため**時間がかかる**。

このMCPサーバーを介することで、**ファイルの読み書き・重複チェック・整形をスクリプト側で一括処理**し、  
メモ書きのスピードを向上させることが目的。

---

## 🧩 背景

### 現状の問題

```
スキル実行
  → Claude が TodoWrite（チェックリスト作成）
  → Claude が Grep（重複チェック）         ← 遅い
  → Claude が Write/Edit（ファイル保存）   ← 遅い
  → 完了
```

Claude が**LLMとして1ステップずつ考えながら動く**ため、
ファイル操作のたびにレスポンス待ちが発生する。

### 改善後のイメージ

```
スキル実行
  → Claude がメモ内容を決める（LLMの仕事）
  → MCPサーバーを1回呼ぶだけ             ← 速い
  → スクリプトが Grep＋Write を一瞬で処理
  → 完了
```

**Claudeが「考える部分」はそのまま**なので精度は落ちない。  
ファイル操作の部分だけをスクリプトに任せることでスピードアップ。

---

## 🔧 システム構成

```
Claude Code（VSCode拡張）
  ↓ スキル実行（/memo-all など）
  ↓ MCPツール呼び出し
MCPサーバー（Python / ローカル起動）
  ↓
ファイル操作（Grep・Write・Append）
  ↓
メモファイル（01_memo.md / 00_mis_log.md など）
```

---

## 📦 必要な機能（MCPツール一覧）

### 1. `memo_write`
- **用途**: メモを指定ファイルに追記する
- **引数**:
  - `file_path`: 書き込み先ファイルパス
  - `content`: 追記する内容（Markdownテキスト）
  - `encoding`: 文字コード（デフォルト: UTF-8）
- **処理**: UTF-8で安全にファイルへ追記

---

### 2. `memo_grep`
- **用途**: メモファイル内の重複チェック
- **引数**:
  - `file_path`: 検索対象ファイルパス
  - `keyword`: 検索ワード
- **戻り値**: マッチした行番号と内容のリスト

---

### 3. `memo_read_last`
- **用途**: ファイルの末尾N行を取得（追記前の確認用）
- **引数**:
  - `file_path`: 対象ファイルパス
  - `lines`: 取得行数（デフォルト: 10）
- **戻り値**: 末尾N行のテキスト

---

### 4. `memo_write_both`
- **用途**: `01_memo.md` と `00_mis_log.md` に同時書き込み（`/memo-all` 用）
- **引数**:
  - `memo_path`: 詳細メモファイルパス
  - `log_path`: 気づき・ミスログファイルパス
  - `memo_content`: 詳細メモの内容
  - `log_line`: 1行ログの内容
- **処理**: 2ファイルへの書き込みをアトミックに実行

## ⚙️ settings.json への登録方法（予定）

Claude Code の `settings.json` に以下を追加することで、  
スキルからMCPツールとして呼び出せるようになる。

```json
{
  "mcpServers": {
    "auto-memo": {
      "command": "python",
      "args": ["D:/01_AI/10_Python/00_Python_src/99_commit/84_auto-memo補助MCPツール/server.py"]
    }
  }
}
```

## 📝 備考

- 文字コードは**UTF-8固定**（PowerShell経由のShift-JIS文字化け問題を回避するため）
- Python の `mcp` ライブラリを使用予定（`pip install mcp`）
- ローカル専用ツールのため認証・セキュリティは最小限でOK
